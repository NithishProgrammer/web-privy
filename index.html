How It Works

1. User Input

"My Aadhaar is [Redacted]."

2. Mask

"Aadhaar is ⟦PII_AADHAAR_x1⟧."

3. AI Model

Processes the masked text in total safety without exposure.

4. Restore

Swaps token back to original.

✨ Features

  • Advanced Detection: Optimized patterns for precise PII identification.
  • Checksum Validation: Verifies mathematical integrity (e.g., Verhoeff).
  • Reversible Masking: Maintain total context without data loss.
  • Smart Tokenization: Structured ⟦PII_TYPE_ID⟧ format for LLMs.
  • Fast & Lightweight: Minimal overhead for real-time applications.
  • No Data Leakage: Ensures sensitive data is never exposed. Data is not Stored in any database. It never leaves your system.

🛠️ Usage

mask_example.py 
import privylayer as pl

# 1. Protect (Mask data)
text = "My phone is 9876543210 and Aadhaar is [Aadhaar Redacted]"
mask, context, seen = pl.mask(text)

print(mask)
# Output: My phone is ⟦PII_PHONE_x1⟧ and Aadhaar is ⟦PII_AADHAAR_x1⟧

# 2. Restore (Unmask data)
original = pl.restore(masked_response, context)

print(original)
# Output: My phone is 9876543210 and Aadhaar is [Aadhaar Redacted]

Limitations

  • No DB Lookup: PAN/IFSC uses structural regex matching, not external banking APIs.
  • Checksum Only: Performs mathematical validation but no biometric/OTP authentication.
  • Regex-Based: May occasionally struggle with dictionary words used as names.
  • Stateless: You are responsible for securely managing the mapping context in your environment.
